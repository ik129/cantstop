# キャントストップ オンライン

## 概要

「キャントストップ オンライン」は、人気のある戦略的ボードゲーム「キャントストップ (Can't Stop)」をブラウザ上でオンラインでプレイできるようにしたものです。プレイヤーはダイスを振り、その出目を使ってボード上のトラックを進み、最初に3つのトラックを占領することを目指します。

## 遊び方 (基本ルール)

*   **プレイヤー数:** 2～4人
*   **ゲームの準備:**
    1.  ホストプレイヤーは「部屋を作成」し、表示された部屋IDを他のプレイヤーに伝えます。
    2.  他のプレイヤーは部屋IDと自分の名前を入力して「部屋に入る」ことで参加します。
    3.  ホストは参加者が揃ったら「ゲームを開始」します。

*   **ゲームの進行:**
    1.  **ダイスロール:** 自分のターンになったら、「ダイスを振る」ボタンで4つの6面体ダイスを振ります。
    2.  **ペア選択:** 4つのダイスを2つずつのペアに分け、それぞれのペアの合計値を計算します。3通りの組み合わせが提示されるので、その中から1つを選択します。
        *   例: ダイスが (1, 2, 3, 4) の場合
            *   (1+2)=3 と (3+4)=7
            *   (1+3)=4 と (2+4)=6
            *   (1+4)=5 と (2+3)=5
    3.  **一時マーカーの配置と進行:**
        *   選択したペアの合計値に対応するトラック（列）に、自分の一時マーカー（現在のターンでのみ有効な仮のマーカー）を配置または進めます。
        *   **新規配置の場合:** そのトラックに自分の確定マーカー（後述）がなければ、トラックの最初のマス（Can't Stopスペース）から1マス進んだ位置（つまり、トラック上の数字「1」のマス）に一時マーカーを置きます。もし自分の確定マーカーが既にあれば、その確定マーカーの**次のマスから**一時マーカーの進行が始まります（例: 確定マーカーが3のマスにあれば、一時マーカーは4のマスに置かれます）。
        *   **進行の場合:** 既にそのトラックに一時マーカーがあれば、それを1マス進めます。
        *   一時マーカーは、異なる3つのトラックまでしか同時に配置できません。4つ目のトラックに進もうとする場合は、進行できません。
        *   既に他のプレイヤーが占領しているトラックには、一時マーカーを配置・進行できません。自分が占領したトラックにもそれ以上進めません。
    4.  **「ストップ」または「続ける」:**
        *   **ストップ:**
            *   現在のターンで進めた全ての一時マーカーの位置に、自分の色の「確定マーカー」（自分の進捗を示すコマ）を置きます（または既存の確定マーカーをその位置まで進めます）。これらの確定マーカーは、次回以降の自分のターンで進行の再開地点となります。
            *   もし一時マーカーがトラックのゴールラインに到達してストップした場合、そのトラックは占領され、「占領マーカー」が置かれます。
            *   ターンを終了し、次のプレイヤーに移ります。
        *   **続ける:** 現在の一時マーカーの位置を維持したまま、再度ダイスを振るところから繰り返します。
    5.  **バスト:**
        *   ダイスを振った後、選択したペアのいずれの合計値でも一時マーカーをルールに従って進めることができない場合、その時点でバストとなります。
        *   バストした場合、そのターンで進めた一時マーカーは全て失われ（確定マーカーの位置には影響しません）、ターンが終了します。

*   **マーカーの種類と表示:**
    *   **一時マーカー:** 現在のターンでのみ有効な進行を示します。通常は半透明の円形で表示されます。
    *   **確定マーカー (進捗マーカー):** ストップ時に置かれ、その列のそこまでの進捗を永続的に示します。プレイヤーカラーの少し小さめのコマで表示されます。
    *   **占領マーカー:** トラックのゴールラインに到達し、ストップすることで置かれるマーカー。そのトラックの制覇を示します。プレイヤーカラーの大きめのコマで表示されます。
    *   *注: 同じマスに複数のマーカーが置かれた場合、視認性のため少しずれて表示されることがあります。*

*   **列の占領:**
    *   プレイヤーが「ストップ」を選択した際に、一時マーカー（結果的に確定マーカーとなる）がトラックのゴールラインに到達していれば、そのトラックはそのプレイヤーによって占領され、占領マーカーが置かれます。
    *   占領されたトラックには、他のプレイヤーはマーカーを進めることができなくなります。

*   **勝利条件:**
    *   いずれかのプレイヤーが最初に **3つのトラックを占領** した時点で、そのプレイヤーの勝利となりゲーム終了です。

## 技術スタック

*   **フロントエンド:** HTML, CSS, JavaScript (Vanilla JS)
*   **バックエンド/データベース:** Firebase
    *   **Firestore:** ゲームの状態管理、リアルタイムデータ同期
    *   **Firebase Hosting:** アプリケーションのホスティング

## セットアップとローカルでの実行方法 (開発者向け)

1.  **必要なもの:**
    *   Node.js (npmを含む)
    *   Firebase CLI (`npm install -g firebase-tools`)
    *   Firebase プロジェクト (Firebaseコンソールで作成)

2.  **Firebaseプロジェクトの設定:**
    *   Firebaseコンソールで新しいウェブアプリを作成し、表示されるFirebase SDKの構成オブジェクト（`firebaseConfig`）を取得します。
    *   リポジトリ内の `firebase-config.js` ファイル（なければ作成）に、取得した `firebaseConfig` の値を正しく記述してください。
        ```javascript
        const firebaseConfig = {
          apiKey: "YOUR_API_KEY",
          authDomain: "YOUR_AUTH_DOMAIN",
          projectId: "YOUR_PROJECT_ID",
          // ... その他の設定値
        };
        ```
    *   Firestoreデータベースを作成し、テストモードで開始するか、適切なセキュリティルールを設定してください（開発初期はテストモード推奨）。

3.  **ローカルでの実行:**
    *   リポジトリをクローンまたはダウンロードします。
    *   `index.html` ファイルをブラウザで直接開くか、VSCodeの「Live Server」拡張機能のようなローカル開発サーバーを使用して起動します。

4.  **Firebase Hostingへのデプロイ:**
    *   Firebase CLIでログイン (`firebase login`)。
    *   プロジェクトルートで `firebase init hosting` を実行し、指示に従って設定します。
        *   既存のFirebaseプロジェクトを選択。
        *   公開ディレクトリとして `.` (カレントディレクトリ) を指定（または `public` フォルダを作成し、そこに全ファイルを移動して `public` を指定）。
        *   シングルページアプリケーションとして設定 (`Yes`)。
        *   (GitHub連携は任意)
    *   `firebase deploy --only hosting` コマンドでデプロイします。
