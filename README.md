# キャントストップ オンラインゲーム

## 概要

「キャントストップ」は、サイコロを使ったクラシックなボードゲームです。プレイヤーはリスクを取りながらトラックを進み、最初に3つのトラックを占領することを目指します。このプロジェクトは、Firebaseを利用してオンラインで複数プレイヤーが対戦できるウェブアプリケーション版です。

## 技術スタック

*   フロントエンド: HTML, CSS, JavaScript
*   バックエンド: Firebase (Firestore, Firebase Hosting)
*   リアルタイム同期: Firestore Realtime Updates

## ゲームの準備

1.  **部屋の作成または参加:**
    *   **部屋を作成:** ホストとして自分の名前を入力し、「部屋を作成」ボタンを押します。部屋が作成されると、画面に**部屋ID**が表示されます。このIDを他の参加者に伝えてください。
    *   **部屋に参加:** 参加者として自分の名前と、ホストから伝えられた部屋IDを入力し、「部屋に入る」ボタンを押します。
2.  **ゲーム開始:**
    *   ホストは、参加者が揃ったら「ゲームを開始」ボタンを押してゲームを開始します。
    *   ロビー画面（ゲーム開始待機画面）では、参加プレイヤーリストと共に部屋IDも表示されるため、後から参加する人に伝える際も便利です。

## 遊び方

### 1. ゲームの目的

*   最初に3つのトラックを占領（ゴールに到達）したプレイヤーが勝者となります。

### 2. マーカーの種類と役割

ゲーム中、3種類のマーカーが使用されます。

*   **一時マーカー (Temporary Markers):**
    *   現在のターンの間だけ、ダイスの合計値に対応するトラック上に置かれるマーカーです（通常は半透明の円形）。
    *   プレイヤーは、ダイスを振るたびに、これらのマーカーを進行させるか、新しいトラックに配置します。
    *   一時マーカーは最大3つまでしか使用できません（3つの異なるトラックに配置できます）。
*   **確定マーカー (Progress Markers):**
    *   プレイヤーが「ストップ」を選択した際に、その時点での一時マーカーの位置に置かれる、各プレイヤーの色が付いたマーカーです。これらは「中間マーカー」とも呼ばれ、そのトラックにおけるプレイヤーの恒久的な進捗を示します（通常はプレイヤーカラーの小さなコマ）。
    *   次回以降のターンで同じトラックを選択した場合、この確定マーカーの位置から進行を再開できます。
*   **占領マーカー (Claim Markers):**
    *   いずれかのトラックのゴールライン（最上部のマス）に到達し、「ストップ」を選択するか、ゴールラインに到達した時点で自動的にトラックが占領された場合、そのトラックのゴールラインに置かれるプレイヤー色のマーカーです（通常はプレイヤーカラーの大きなコマ）。
    *   トラックを占領すると、他のプレイヤーはそのトラックを使用できなくなります。

### 3. 自分のターン

*   自分のターンが来ると、まず「ダイスを振る」ボタンを押して4つのサイコロを振ります。
*   サイコロの目は2つずつペアにして、合計2つの「合計値」を作ります。
    *   例: ダイスが (1, 2, 3, 4) の場合、(1+2)=3 と (3+4)=7 や (1+3)=4 と (2+4)=6 などの組み合わせが可能です。
*   有効な組み合わせがUIに表示されるので、1つ選択します。
*   **自動バスト:** もし、ダイスロール後、提示されたどの組み合わせを選んでも、ルール上1マスもマーカーを進めることができない場合は、自動的に「バスト」となり、ターンが終了します。

### 4. マーカーの進行と配置

*   選択した2つの合計値に対応するトラックに一時マーカーを配置または進行させます。
    *   **新規配置の場合:** そのトラックに自分の確定マーカーがなければ、トラックの最初のマス（Can't Stopスペース）の次のマス（位置1）から一時マーカーの進行が始まります。もし自分の確定マーカーが既にあれば、その確定マーカーの**次のマスから**一時マーカーの進行が始まります。
    *   **進行の場合:** 既にそのトラックに一時マーカーがあれば、それを1マス進めます。
*   **3マーカー制限:** 一時マーカーは同時に3つの異なるトラックまでしか配置できません。
    *   既に3つのトラックに一時マーカーがある場合、選択した合計値のトラックがその3つのうちのいずれかでないと進行できません。
*   **3つ目マーカーの選択ルール (特別ルール):**
    *   一時マーカーが既に2つ盤上にあり、かつダイスの組み合わせで選択した2つの合計値S1, S2が、両方ともまだ一時マーカーが置かれていない新しいトラックで、かつ両方とも配置可能な場合、プレイヤーには「S1のトラックに配置」と「S2のトラックに配置」という2つの選択肢が提示されます。プレイヤーはどちらか一方を選び、選んだトラックにのみ3つ目の一時マーカーが置かれます。
    *   上記以外のケース（例: S1かS2の一方または両方が既に一時マーカーのあるトラック、S1とS2が同じ値など）では、システムがルールに従って自動的にマーカーを配置/進行します。
*   進行先のマスがゴールラインの場合、そのトラック上で一時マーカーはゴールラインに到達します。

### 5. 「続ける」か「ストップ」か

マーカーを配置/進行させた後、以下の選択ができます。

*   **続ける (Continue):**
    *   再度「続ける」ボタン（元「ダイスを振る」ボタン）を押してサイコロを振り、さらにリスクを取ってトラックを進めることができます。
    *   この際、新たに選択するダイスの合計値のトラックに一時マーカーを進められない場合、または有効な組み合わせがない場合は「バスト」となります。
*   **ストップ (Stop):**
    *   「ストップ」ボタンを押すと、現在のターンを終了します。
    *   この時、全ての一時マーカーの位置に、自分の色の「確定マーカー」が置かれます（または既存の確定マーカーをその位置まで進めます）。
    *   もし一時マー커がトラックのゴールラインに到達していた場合、そのトラックはあなたのものとなり、「占領マーカー」が置かれます。
    *   ターンが次のプレイヤーに移ります。

### 6. バスト (Bust)

以下の場合、プレイヤーは「バスト」となり、そのターンに達成した一時的な進捗は全て失われます。

*   ダイスを振った後、どの組み合わせを選んでも一時マーカーを1マスも進めることができない場合（**自動バスト**）。
*   「続ける」を選択してダイスを振り直した後、選択した組み合わせで一時マーカーを1マスも進めることができない場合。
*   バストした場合、全ての一時マーカーは盤上から取り除かれ（確定マーカーの位置には影響しません）、ターンが終了します。

### 7. ゲームの終了

*   いずれかのプレイヤーが最初に **3つのトラックを占領** した時点で、そのプレイヤーの勝利となりゲームは終了します。

## 技術スタック
*   フロントエンド: HTML, CSS, JavaScript (Vanilla JS)
*   バックエンド/データベース: Firebase (Firestore, Firebase Hosting)
*   リアルタイム同期: Firestore Realtime Updates

## セットアップとローカルでの実行方法 (開発者向け)
1.  Firebaseプロジェクトを作成し、ウェブアプリを追加して `firebaseConfig` を取得。
2.  `firebase-config.js` に `firebaseConfig` を設定。
3.  Firestoreデータベースを有効化 (テストモード推奨)。
4.  `index.html` をブラウザで開くか、ローカルサーバーでホスト。
5.  (デプロイする場合) Firebase CLI をセットアップし `firebase deploy --only hosting` を実行。

詳細はコード内のコメントや、各ファイルの役割を参照してください。
